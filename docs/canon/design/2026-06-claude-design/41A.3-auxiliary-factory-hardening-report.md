## 3. Capability Matrix (SPEC-002 Apêndice B)
**Orquestrador:** modelo, prompt, HTTP tools, MCP, memória/RAG, segurança, chat direto, **Widget, WhatsApp, Especialistas, Commerce**. **SubAgent Especialista:** modelo, prompt, HTTP tools, MCP, **memória/RAG, segurança**, chat direto só para **debug**; **sem** Widget/WhatsApp/Especialistas/Commerce. Orquestrador conduz; especialista executa tarefa técnica.

## 4. Abas de Agent/SubAgent: decisão tomada
**Decisão: Memória e Segurança são exibidas também para SubAgents** — subagents são `agents` com as **mesmas colunas** (`memory`/`security_settings`), então é tecnicamente seguro e necessário (um especialista de apólice/sinistro precisa de RAG e guardrails). Removidos os gates `!isSubagent` apenas dessas duas abas em `AgentConfigModal`. **Mantidos ocultos** para SubAgent: Widget, WhatsApp, Commerce, Especialistas. Microcopy do switch atualizada: "Especialistas… podem usar Memória/RAG e Segurança, mas não têm canais públicos próprios."

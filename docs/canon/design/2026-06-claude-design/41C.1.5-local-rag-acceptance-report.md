## 0. Escopo
Batch de **trava de qualidade** (aceitação/regressão/documentação) do RAG local do Smith antes do conhecimento global. Entregáveis: checklist canônico, SQL SELECT-only, este relatório, e **uma** adição segura e neutra ao `/rag-health` (sinais estáticos). Nenhuma mudança de comportamento de retrieval.

## 1. O RAG local está usando o runtime Smith existente?
**Sim.** Auditoria do código confirma o caminho único do Smith:
- Upload: `documents.py POST /upload` → `DocumentService.upload_document` (DB `public.documents` + MinIO bucket `documents`) → `ingestion_service` (chunk + embeddings `text-embedding-3-small` 1536 + BM25 sparse) → `qdrant_service.insert_embeddings` (coleção `company_{company_id}`).
- Retrieval: `graph._build_initial_state` (prefetch 41C.1.2) e tool `knowledge_base_search` → `SearchService.smart_search` → `qdrant.search_similar`. Nenhum fluxo alternativo.

## 2. Existe motor paralelo?
**Não.** Há um único `SearchService` + `QdrantService`. O `langchain_service.search_documents/get_rag_context` é **legado/dead** (sem callers no chat; confirmado por scan). Os batches 41C.1.2–41C.1.4 só adicionaram **prefetch**, **lexical rescue** e **fallback de score** dentro do mesmo serviço — não um motor novo.

## 3. Isolamento por company_id e agent_id preservado?
**Sim.**
- **company_id:** isolado pelo **nome da coleção** Qdrant `company_{company_id}` (cada empresa tem coleção própria) — não há mistura entre empresas.
- **agent_id:** salvo no **payload** de cada chunk e aplicado como **filtro** em `search_similar` (prefetch e tool injetam `agent_id`). `smart_search`/`_execute_search` repassam `agent_id`. Teste C do checklist valida o não-vazamento A↔B.

## 4. Tenant-wide docs preservados?
**Sim.** `_execute_search` chama `search_similar(..., include_tenant_wide=True)`: o agente vê **seus docs + docs tenant-wide** (`agent_id` nulo), **nunca** de outro agente. Inalterado desde 41C.1.

## 5. include_global continua OFF?
**Sim.** Default `False` em `smart_search`, `_execute_search`, `langchain_service.*`. Os **únicos** callers (`documents.py` rag-debug e `graph.py` prefetch) passam **`False`** explicitamente. Scan: **nenhum** `include_global=True`. `/rag-health` agora expõe `include_global_default=false` como sinal de aceitação.

## 1. Resumo executivo
Completa a experiência da Fábrica: agora dá para **publicar um Agent/Subagent existente como Auxiliar** (global ou exclusivo), com **blueprint sanitizado** (sem segredos). O **GET legado de integrations** deixou de retornar token/client_token (só flags). A **SPEC-002** ganhou um apêndice detalhado (fluxo completo, global×local, tipos, RAG, corredores, regras). O **tenant dashboard** abre Auxiliares com motor inteligente numa página de detalhe segura (sem "Em breve" quando há agent).

## 2. O que foi implementado
- **Publicar Agent existente** (`/admin/auxiliares` → botão + modal) + API `POST /api/admin/auxiliaries/templates/from-agent`.
- `extractBlueprintFromAgent()` + `fetchAgentViaBackend()` (helpers seguros).
- **Strip de token** no GET `/api/admin/integrations` (flags em vez de valores).
- **SPEC-002** apêndice A (A1–A8).
- **Detalhe genérico** `/dashboard/auxiliares/galeria/[slug]` + Meus "Abrir" para qualquer Auxiliar instalado.

## 3. Como publicar Agent existente
Modal: seleciona **empresa** → carrega agents da empresa (reuso `GET /api/admin/agents/company/[companyId]`, já sanitizado) → seleciona **Agent/Subagent** (prefill nome/slug) → nome, slug, categoria, descrições, **visibilidade** (global/exclusivo), **status**, e **"instalar na empresa (vincula agent original)"**. Submete a `from-agent`.

## 4. Como o blueprint é extraído
`extractBlueprintFromAgent(agent)`: whitelist de campos **não sensíveis** (name, slug, is_subagent, allow_direct_chat, llm_provider, llm_model, agent_system_prompt, llm_temperature/max_tokens, allow_web_search/vision, is_hyde_enabled, tools_config, security_settings, widget_config, retrieval_mode, personality, reasoning_effort, verbosity) → `sanitizeBlueprint()` (remoção profunda de qualquer chave proibida). Guardado em `auxiliary_templates.default_config.runtime.agent_blueprint`.

## 5. Como segredos são removidos
- O agent vem do backend já **sanitizado** (`has_api_key`/`has_vision_api_key`/`has_whatsapp` são flags; **sem chaves cruas**).
- `BLUEPRINT_ALLOWED_KEYS` **não** inclui campos de segredo; `sanitizeBlueprint` remove `FORBIDDEN_SECRET_KEYS` em qualquer nível.
- Nada de `integrations`/Vault/tokens entra no blueprint.

## 6. Global × exclusivo
`default_config.visibility = { type:'global' }` ou `{ type:'private', company_id }`. Marcado no template para filtragem futura na Galeria. No MVP, exclusivo pode ser instalado direto na empresa (e o template fica marcado private).

## 7. Instalação: vincula original ou cria cópia
- **Publicar com "instalar na empresa de origem"** → `tenant_auxiliaries.config.runtime = { kind:'smith_agent', agent_id:<original>, created_from_template:true, linked_original:true }` — **vincula o agent original, não duplica**.
- **Instalar o template em OUTRA empresa** (fluxo 41A.1) → cria um **novo** agent a partir do blueprint.
- **Idempotência** por `(company_id, slug)` mantida.

POST/PATCH já bloqueavam escrita (39A4.1). O modal legado continua funcional (campos de segredo apenas não pré-preenchem).

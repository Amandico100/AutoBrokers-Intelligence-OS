## 4. Seletor de agente (corrigido)
Adicionada prop `showAgentSelector` (default `true`) em `InputArea` e `AnimatedAIChat`. A renderização do bloco do seletor agora exige `showAgentSelector && agents.length > 0 && onAgentChange`. O chat tenant passa `showAgentSelector={false}` → **"JARVYS Sandbox" não aparece mais** no composer (desktop/mobile). A lógica de agentes (`agents`, `selectedAgentId`, `onAgentChange`, troca de agente, stream) **foi preservada** — apenas oculta na UI do corretor.

## 5. Composer mobile (corrigido)
Causa do overflow: seletor de agente (texto longo) + botões + "Enviar" com texto. Correções: seletor oculto (§4); linha inferior com `flex-wrap` e `gap-2 sm:gap-4`; "Enviar" vira só ícone no mobile (texto `hidden sm:inline`). Voz, anexo, web-search e envio mantidos. Em ~360–430px o botão não sai mais da caixa.

## 6. Bottom nav mobile (corrigido)
Causa: `h-screen` (100vh) inclui o chrome do navegador móvel, empurrando a bottom nav para baixo da dobra. Corrigido com `h-[100dvh]` (dynamic viewport height) no `TenantAppShell`. A bottom nav segue **no fluxo** (não cobre o composer) e agora aparece **sem precisar rolar**. Desktop inalterado.

## 7. Frase principal (ajustada)
`ChatWelcome`: wrapper com `px-2` (somado ao `px-4` do container = respiro lateral) e h1 com `max-w-[20rem]` + `leading-tight` no mobile (`sm:max-w-none sm:text-3xl`). Frase e BrandMark mantidos.

## 8. "Nova conversa" determinística
Implementado via **evento client-side** (sem backend, sem query param, sem Suspense): `TenantNav`/`TenantTopBar` disparam `window.dispatchEvent(new CustomEvent('autobrokers:new-conversation'))` quando o usuário já está em `/dashboard` ou `/dashboard/chat`; caso contrário `router.push('/dashboard')` (o `ChatPage` monta com nova sessão). O `ChatPage` escuta o evento e chama `handleNewConversation()` (reseta `sessionId`, `conversationId`, `messages`). Drawer mobile fecha via `onNavigate`. **Optei por evento em vez de query param** para evitar `useSearchParams`/Suspense e manter `/dashboard` estático.

## 9. Diagnóstico read-only — "Não sei" para Roma
**Conclusão: é problema de DADOS/CONFIG do agente ativo (banco), não de frontend nem de restrição no código.**
- **Seletor mostra "JARVYS Sandbox" = dado residual.** Não há "JARVYS" em código (`app/`, `backend/`, `lib/`). O bootstrap atual (`app/api/admin/sandbox/bootstrap-tenant/route.ts`) cria um agente **"AutoBrokers Sandbox"** (slug `autobrokers-sandbox`) com prompt benigno: *"Você é o AutoBrokers Sandbox… responda de forma curta… diga que ainda está em modo sandbox."* Ou seja, a empresa de sandbox ainda tem um **agente antigo "JARVYS Sandbox"** (pré-rename) na tabela `agents`, e `/api/agents` o retorna; `selectedAgentId = agents[0]` o usa.
- **O código NÃO força RAG-only / "não sei".** Ao contrário: `backend/app/core/prompts.py:29` diz *"Só diga 'não sei' se realmente esgotar as opções"* e `backend/app/agents/nodes.py:162` diz *"Não responda 'não sei' sem antes buscar nos documentos"*. Logo, o RAG **não** está bloqueando conhecimento geral por código.
- **Causa provável do "Não sei":** o `agent_system_prompt` (coluna em `agents`) do agente ativo residual ("JARVYS Sandbox") é antigo/placeholder/restritivo, e/ou o **modelo/provedor/LLM key** daquele agente está ausente/mal configurado (o `agent_system_prompt` é o que o `nodes.py` injeta como `SystemMessage`).
- **Onde ajustar depois (sem SQL agora):** tabela **`agents`** — corrigir/limpar o agente ativo da empresa sandbox (nome, `agent_system_prompt`, colunas de modelo/provedor/credencial), OU re-rodar `POST /api/admin/sandbox/bootstrap-tenant` e garantir que o agente ativo seja o "AutoBrokers Sandbox" (slug `autobrokers-sandbox`). Atenção: o residual pode ter slug/nome diferente e coexistir.
- **SQL diagnóstico sugerido (NÃO executado):**
  `select id, name, slug, is_active, agent_system_prompt, /* colunas de modelo/provedor */ from agents where company_id = '<sandbox_company_id>' order by created_at;`
  (Verificar qual agente está ativo, seu prompt e se há modelo/credencial configurados.)
- **Comportamento-alvo:** usar conhecimento geral da LLM em perguntas gerais; usar RAG quando depender de dados internos; dizer que não encontrou o dado **específico** quando faltar — nunca "não sei" para conhecimento geral, salvo prompt explícito.

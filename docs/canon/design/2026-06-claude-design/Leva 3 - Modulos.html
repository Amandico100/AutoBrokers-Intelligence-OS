Resumo de Atendimentos

Disponível
Atendimento · Lê conversas e gera um resumo — sem envio externo
Visão geral
Configuração
Conectores
Permissões
Execuções

Recebe uma conversa ou caso e devolve um resumo claro: o que aconteceu, quais pendências existem e qual a próxima ação sugerida. Ideal para o corretor entender um atendimento em segundos, sem reler tudo.

O que ele faz
Lê a conversa selecionada (sandbox no início).
Gera resumo, pendências e próxima ação.
Salva a execução com custo e histórico.
Como começa

Execução manual sobre uma conversa que você escolher. Sem disparo automático, sem mensagem ao cliente. Quando ganhar confiança, evolui para rodar sozinho — sempre dentro das permissões.

Fechamento das levas · pronto para handoff

Com as três levas aprovadas, o Dashboard da Corretora tem fundação, padrões e módulos definidos. O próximo passo é o handoff para o Claude Code — specs fechadas por tela, marcando o que é shell visual e o que conecta a dados reais.

SHELL VISUAL

Auxiliares (lista/galeria/detalhe), página de Seguradora, lista e detalhe de Caso — sem backend novo, sem lógica de corredor.

DADOS REAIS

Auxiliar de Resumo de Atendimentos: execução manual sobre conversa selecionada, salva histórico e custo, sem envio externo.

HITL

Toda ação externa (acionar seguradora, enviar mensagem) exige estado “Aguardando aprovação” na UI antes de executar.

ORDEM

chat-first → sidebar → shells → Auxiliar de Resumo → base de Atendimento, alinhado ao ROADMAP.

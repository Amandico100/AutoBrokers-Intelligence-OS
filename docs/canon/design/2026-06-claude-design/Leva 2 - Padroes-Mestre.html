Leva 2 · Os três padrões reutilizáveis

Três padrões. Todo o produto.

Sobre a fundação aprovada, desenho aqui as três peças que se repetem em cada módulo — Galeria, Página de Detalhe e Modal de Permissão — mais o sistema de estados que percorre tudo. Construídos uma vez, muito bem, e reutilizados em Auxiliares, Conectores, Seguradoras e Atendimentos. É o que faz o produto inteiro parecer coeso.

02

Padrão-Mestre B · Página de Detalhe

progressive disclosure

Cabeçalho com ícone, nome, status e ação primária. Corpo em abas que revelam a complexidade aos poucos: Visão geral → Permissões → Conexões → Histórico. Sempre com caminho de volta. Aqui um conector; a mesma anatomia serve a Auxiliar, Seguradora e Caso.

B1

Cabeçalho fixo

Ícone, nome, status e UMA ação primária. Ações secundárias ficam discretas dentro das abas.

B2

Abas = camadas

Cada aba revela só o necessário. Nada de uma página única e gigante com tudo aberto.

B3

Linguagem humana

“Vai poder ler seus arquivos”, não “OAuth scope”. O painel lateral resume risco e quem usa.

B4

Breadcrumb no desktop, voltar contextual no mobile. O usuário nunca se perde nas camadas.

03

Padrão-Mestre C · Modal de Permissão

consentimento · HITL

Aparece ao conectar algo ou autorizar uma ação. Mostra com clareza o que será acessado, para quê, qual o risco e quem poderá usar — em linguagem de corretora. Para ações sensíveis, comunica que haverá aprovação humana antes de qualquer execução externa.

C4

Modal só para o curto

Conectar, confirmar, aprovar. Configuração longa vira página (B), nunca modal gigante.

um sistema, não 16 telas

06

Onde cada padrão se repete

A promessa da Leva 2: desenhados uma vez, aplicados em todo lugar na Leva 3. Esta é a matriz de reuso que entrego para o Claude Code implementar como componentes.

A Galeria

  • Galeria de Auxiliares (modelos prontos)
  • Galeria de Conectores
  • Lista de Seguradoras

B Página de Detalhe

  • Detalhe de Auxiliar + ativação
  • Detalhe de Conector
  • Página de Seguradora (abas)
  • Detalhe de Caso (Atendimentos)

C Modal de Permissão

  • Conectar um conector
  • Autorizar um Auxiliar
  • Aprovar ação de atendimento (HITL)
Próximos passos · aguardando sua aprovação

Aprovados os padrões-mestre e os estados, sigo para a Leva 3 — Módulos: Auxiliares (com o Auxiliar de Resumo de Atendimentos), Personalização → Conectores, Personalização → Seguradoras e o shell de Atendimentos — todos aplicando A, B, C e o sistema de status definidos aqui.

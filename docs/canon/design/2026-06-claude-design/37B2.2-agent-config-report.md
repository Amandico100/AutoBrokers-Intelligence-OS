## 1. Resumo executivo
O chat tenant seleciona o **agente de chat ativo mais antigo** (ordenação `created_at ASC`, `agents[0]`). Existe um agente residual **"JARVYS Sandbox"** (anterior ao rename) que, por ser mais antigo, é escolhido antes do "AutoBrokers Sandbox" criado pelo bootstrap — daí o nome residual no composer e o "Não sei" (prompt antigo/placeholder). A correção é de **dados/config na tabela `public.agents`**, não de frontend nem de código. Entreguei: SQL de diagnóstico (SELECT), SQL de limpeza (UPDATE idempotente, sem DELETE) e este relatório. **Nada foi executado.**

## 2. O problema observado
No deploy do 37B2.1: visual correto, mas ao perguntar "Onde fica a cidade de Roma?" o agente respondeu **"Não sei"** — comportamento incompatível com uma LLM geral. Indício de agente ativo com prompt/dado antigo.

## 3. Diagnóstico do fluxo de escolha do agente
- `app/dashboard/chat/page.tsx`: `fetch('/api/agents')` → `setAgents(data.agents)` → `selectedAgentId = data.agents[0].id` (o primeiro da lista).
- `app/api/agents/route.ts`: descobre a empresa do usuário (`users_v2.company_id`), e:
  `from('agents').select('id, name, is_subagent, allow_direct_chat').eq('company_id', …).eq('is_active', true).order('created_at', { ascending: true })`, depois filtra `!is_subagent || allow_direct_chat`.
- **Conclusão:** `agents[0]` = **agente de chat ativo MAIS ANTIGO** da empresa. Um "JARVYS Sandbox" antigo vence o "AutoBrokers Sandbox" novo (bootstrap).
- O backend (`/chat/stream` → FastAPI) recebe `agentId` e usa `agents.agent_system_prompt` como `SystemMessage` (campo confirmado em `backend/app/agents/nodes.py`, `graph.py`, `models/agent.py`, `api/agents.py`). **O `agents` é a fonte autoritativa.**

## 4. Estrutura real da tabela `agents` (schema_completo.sql, linha 444)

- **Legado:** `public.companies` também tem `agent_system_prompt`, `llm_provider`, `llm_model`, `agent_enabled` (modelo antigo, fallback). Tratado como secundário.

## 5. Causa provável do "Não sei" para Roma
1. **Agente residual JARVYS é o ativo** (mais antigo) → seu `agent_system_prompt` antigo/placeholder/restritivo é usado. **(causa mais provável)**
2. Secundário: o agente ativo pode ter `llm_model`/`llm_provider` ausentes/mal configurados (fallback gera resposta pobre).
3. **NÃO é restrição de código:** ao contrário — `backend/app/core/prompts.py:29` ("Só diga 'não sei' se realmente esgotar as opções") e `backend/app/agents/nodes.py:162` ("Não responda 'não sei' sem antes buscar nos documentos") **desencorajam** o "não sei". O RAG não bloqueia conhecimento geral por código.
4. **NÃO é frontend** (o chat só faz stream).

BATCHES-001
Cards de tarefa · para o Claude Code executar

Sete batches fechados. Um PR cada.

Tradução direta das fichas §4 e da ordem §5 do HANDOFF-001 em cards executáveis. Cada card é um escopo fechado: arquivos permitidos, dependências, componentes, estados, shell × dado, fora de escopo, HITL e critério de PRONTO. O Claude Code implementa dentro do card — não decide produto, não cria rota fora do HANDOFF, não troca stack nem paleta.

Stack Next.js + Tailwind + Radix/ShadCN Paleta Névoa Fonte Geist Idioma PT-BR 7 batches + anexo de ícones

Como usar cada card

Abra um PR por card. Cole o card inteiro como instrução. O Claude Code só toca nos Arquivos permitidos. Ao terminar, validar contra o critério de PRONTO e comparar com o mockup da Leva citada.

Regras invioláveis

Não criar rotas/módulos fora do HANDOFF-001. Não trocar stack, paleta ou tipografia. Tokens (B0) entram antes de tudo. Shells antes de dados reais. Ação externa só com estado HITL visível.

Convenção de arquivos

UI em components/ui/, padrões em components/patterns/, layout em components/layout/, rotas em app/ (App Router). Sandbox visual em app/_sandbox/.

→

Ordem & dependências

Sequência obrigatória. Cada batch depende do anterior. O anexo de ícones (B-ICONS) entra junto do B0 — é pré-requisito visual para não inventar ícones.

B-ICONS+ B0 tokens→ B1 shell nav→ B2 chat→ B3 padrões→ B4 shells→ B5 Resumo→ B6 atendimento

Leitura: B-ICONS e B0 podem ser feitos em paralelo, mas ambos precisam estar prontos antes do B1. Do B1 em diante é estritamente sequencial.

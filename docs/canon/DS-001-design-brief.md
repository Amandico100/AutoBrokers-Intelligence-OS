# DS-001 - Design Brief

Status: brief for Claude Design
Product: AutoBrokers.ai
Last updated: 2026-06-06

## 1. This Is Not the Final Design System

This document is a design brief, not the final design system.

Claude Design should later create the definitive visual system, component rules, responsive behavior, tokens and interaction specs.

Do not create `DS-001-design-system.md` until the actual design system exists.

## 2. Product Feeling

AutoBrokers.ai should feel:

- clean;
- modern;
- trustworthy;
- intelligent;
- operational;
- focused;
- premium but not decorative;
- fast to scan;
- calm enough for daily insurance work.

The product should not feel like a generic analytics dashboard or a marketing landing page.

## 3. Visual Direction

Preferred direction:

- dark-first;
- black, white and gray as the base;
- color used with restraint for state, risk, success and brand accents;
- high contrast;
- precise spacing;
- simple surfaces;
- polished typography;
- no excessive gradients;
- no oversized decorative hero sections inside the app.

## 4. Chat-First Surface

The first tenant surface should prioritize:

- AutoBrokers identity;
- central input;
- focused empty state;
- minimal shortcuts;
- strong mobile behavior;
- low visual noise.

The design should be inspired by ChatGPT and Claude without copying either product.

## 5. Mobile-First

The product must work for brokerages that operate from phones and laptops.

Mobile rules:

- primary actions reachable;
- no overloaded sidebars;
- progressive disclosure;
- readable text;
- no dense card grids on the first screen;
- forms and permission screens must be usable on small screens.

## 6. Pages in Layers

For complex modules, use layered flows:

```txt
gallery -> detail -> setup -> permissions -> review -> run/history
```

This is especially important for Auxiliares, connectors and future Atendimento packages.

## 7. Cards

Cards may be used for repeated items, galleries, lists and compact summaries.

Avoid:

- card-heavy home pages;
- dashboard walls;
- decorative nested cards;
- oversized empty cards with low information value.

## 8. Information Density

Start simple. Reveal complexity only when the user enters a module.

Avoid dumping:

- all operational metrics on the home;
- all modules in the sidebar too early;
- every configuration on one page;
- long explanatory text inside the app.

## 9. Handoff to Claude Design

Claude Design should define:

- final theme;
- typography;
- component tokens;
- page templates;
- navigation behavior;
- responsive rules;
- empty states;
- Auxiliares gallery/detail/permissions flow;
- tenant chat-first UX;
- Admin Global refinements.
